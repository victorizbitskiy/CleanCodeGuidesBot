## SOLID

### What is SOLID?

**SOLID** is the mnemonic acronym introduced by Michael Feathers for the first five principles named by Robert Martin, which meant five basic principles of object-oriented programming and design.

- [S: Single Responsibility Principle \(SRP\)](https://github.com/thangchung/clean-code-dotnet/edit/master/README.md#solid)
- [O: Open/Closed Principle \(OCP\)](https://github.com/thangchung/clean-code-dotnet/edit/master/README.md#solid)
- [L: Liskov Substitution Principle \(LSP\)](https://github.com/thangchung/clean-code-dotnet/edit/master/README.md#solid)
- [I: Interface Segregation Principle \(ISP\)](https://github.com/thangchung/clean-code-dotnet/edit/master/README.md#solid)
- [D: Dependency Inversion Principle \(DIP\)](https://github.com/thangchung/clean-code-dotnet/edit/master/README.md#solid)
